## 1. 项目定位

这是一个面向 Rockchip RK3588 平台的视频流 AI 分析项目，核心目标是：

- 多路 RTSP/本地视频流拉取
- 使用 RK MPP 硬解码 H.264/H.265
- 使用 RKNN 在 NPU 上运行目标检测模型
- 使用 RGA 做图像缩放/格式处理加速
- 多模型并行推理并融合检测结果
- 将多路结果拼接成宫格画面
- 使用 MPP 硬编码 + FFmpeg 封装推送到 RTMP/RTSP

整体是典型的“拉流解码 → 缓冲 → 推理 → 结果融合/绘制 → 拼图 → 编码推流”流水线。

## 3. 核心数据流

```text
多路输入视频/RTSP
  ↓
StreamLoaderManager 创建每路 StreamLoader
  ↓
FFmpeg avformat 拉流/读包
  ↓
MPP Decoder 硬解码
  ↓
每路 Mbuffer 保存最新帧
  ↓
每路 rknn_infer 线程读取帧
  ↓
单路内部 ThreadPool 并行运行 person/helmet/callplay 等 RKNN 模型
  ↓
DetectionFusionManager 融合多模型检测结果
  ↓
把融合框画到图像上，写入 images[i]
  ↓
combineImage 按 2 列 × 最多 3 行拼接成大图
  ↓
StreamingManager 队列
  ↓
MPP Encoder H.264 硬编码
  ↓
FFmpeg 封装推送 RTMP/RTSP
```

## 4. 关键模块

### 4.1 StreamLoader / StreamLoaderManager

位置：

- `demo_multhread_decode_infer_mulmodel/include/stream_loader.h`
- `demo_multhread_decode_infer_mulmodel/lib/stream_loader.cpp`

职责：

- 管理多路流地址 `urls[id]`
- `load_stream(id)` 为每一路创建 `StreamLoader` 和解码线程
- 使用 FFmpeg 打开 RTSP/本地文件
- 读取 AVPacket
- 调用 MPP Decoder 输出 BGR/OpenCV 图像
- 将最新帧写入每一路 `Mbuffer buffer`
- 支持 EOF/reconnect 逻辑，源代码里也有本地文件按源 FPS 限速的处理

### 4.2 MPP Decoder

位置：

- `include/mpp_decoder.h`
- `lib/mpp_decoder.cpp`

职责：

- 使用 Rockchip MPP 初始化 H.264/H.265 解码器
- 接收码流包并输出帧
- 回调到 StreamLoader 写入 buffer

### 4.3 rknn_lite / RKNN 推理

位置：

- `include/rknnPool.hpp`

职责：

- 加载 `.rknn` 模型
- 绑定 RK3588 NPU core：`RKNN_NPU_CORE_0/1/2`
- 设置输入图像，执行 `rknn_run`
- 获取输出，调用 `post_process`
- 输出 `detect_result_group_t`

当前多模型版本中主要模型：

- person：人员检测，class_num=1，模型 id=0
- helmet：安全帽检测，class_num=2，模型 id=1
- callplay：打电话/玩手机，class_num=2，模型 id=3
- tired：疲劳检测，代码预留但主流程中 `ptr3 = nullptr`

### 4.4 rknn_infer 单路推理线程

位置：

- `demo_multhread_decode_infer_mulmodel/main.cpp`

职责：

- 每一路视频一个推理线程
- 从该路 `buffer.img` 读取最新帧
- 将同一帧复制给多个模型对象
- 内部用 `ThreadPool(4)` 并行跑多个模型
- 每 `INFER_INTERVAL=2` 帧做一次完整推理，中间帧复用上一帧融合结果
- 调用 `DetectionFusionManager` 融合结果并绘制
- 输出到 `images[i]`

### 4.5 DetectionFusionManager

位置：

- `include/detection_fusion_manager.h`
- `lib/detection_fusion_manager.cpp`

职责：

- 把 person/helmet/tired/callplay 检测结果转换为统一的 `FusedDetection`
- 使用 IoU / IoM 阈值进行融合
- 支持 NMS、置信度阈值、模型权重
- 绘制融合后的框与标签

默认配置要点：

- `iou_threshold = 0.5`
- `iom_threshold = 0.3`，用于 Person + Helmet 这类嵌套关系
- `confidence_threshold = 0.3`
- `enable_weighted_fusion = true`
- `enable_nms = true`

### 4.6 combineImage 拼图线程

位置：

- `demo_multhread_decode_infer_mulmodel/main.cpp`

职责：

- 创建大图画布，当前写死 `1280×1080`
- 每路图缩放成 `640×360`
- 布局为每行 2 路，最多 6 路
- 优先用 RGA `imresize`，失败则回退 OpenCV `cv::resize`
- 控制输出约 24 FPS
- 将拼接后的图送入 `StreamingManager`

注意：当前 `StreamingConfig` 里 width/height 设置为 `1280×720`，但拼图画布是 `1280×1080`，这可能是后续需要修复的问题。

### 4.7 StreamingManager / MPP Encoder

位置：

- `include/streaming_manager.h`
- `lib/streaming_manager.cpp`
- `include/mpp_encoder.h`
- `lib/mpp_encoder.cpp`

职责：

- 管理推流队列，队列超过 10 帧时丢弃旧帧
- 启动独立推流线程
- 使用 MPP Encoder 做 H.264 硬编码
- 用 FFmpeg 写封装并推送 RTMP
- RTSP 相关接口有预留
- 统计 frames_sent / frames_dropped / fps

当前主程序写死 RTMP：

```cpp
stream_config.rtmp_url = "rtmp://192.168.137.1/live/livestream";
```
